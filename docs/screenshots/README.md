# Screenshots

Reference imagery for the project. Primary use: capturing how the
in-game Build Planner renders `.build` files, since GGG's developer
docs describe the data format but not the rendering surface.

## What goes here

- **GGG reveal stills** — frames from official livestreams / videos
  showing the Build Planner UI.
- **In-game captures** — screenshots taken from a live game session
  with a `.build` file loaded.
- **UI mockups** — when we start prototyping our own web app, draft
  comps can live here too.

## Naming convention

`<source>-<short-description>.<ext>`

Examples:
- `reveal-build-list-overview.png`
- `reveal-passive-tree-allocation.png`
- `reveal-skill-gem-tooltip.png`
- `ingame-item-slot-hint-rendering.png`
- `mockup-converter-landing-page.png`

Use kebab-case. PNG preferred for crisp UI screenshots; JPEG fine for
photos of monitors. No hard size limit but please keep individual files
under ~2 MB if practical (compress with a tool like `pngquant` or
TinyPNG if needed).

## Where they're referenced

`docs/ui-exploration.md` Section 2 ("In-game appearance") should link
to specific screenshots once they exist, replacing the
**[exploratory — needs user input]** notes about uncertain rendering
behavior.
